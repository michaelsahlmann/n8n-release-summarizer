# Contributing

Thanks for helping improve this project.

## Getting Started

1. Fork the repository.
2. Create a new branch for your change.
3. Run `npm install`.
4. Copy `.env.example` to `.env` and add your own local keys if needed.
5. Make your changes.
6. Run `npm test`.
7. Open a pull request.

## Security

Never commit real API keys, tokens, credentials, or private `.env` files.

If you accidentally commit a secret, revoke it immediately before opening a pull request.

## Pull Requests

Please keep pull requests focused on one change when possible.

Before opening a pull request, check:

- Tests pass with `npm test`
- No real secrets are included
- Generated release files are intentional
- Documentation is updated when behavior changes

## Code Style

Follow the existing project style. Keep code simple and easy to read.
